# 🚨 Signal Watch — “신호 세기 높은 테크 이슈”

## 📱 앱 개념 & 목표
- **대상:** 바쁜 창업자 · PM · 투자자  
- **핵심 기능:** 특정 토픽(예: *AI 멀티모달*, *전기자전거 보조금*)의 **핵심 신호만** 푸시/이메일로 전달  
- **차별점:** 단순 키워드 알림이 아닌 **정제된 고정밀 이벤트 감지**

---

## ⚙️ 기술 스택
- **프론트:** React Native + Expo  
- **백엔드:** Firebase (Auth / Firestore / FCM)  
- **서버:** Cloud Run (정기 폴링, 룰 기반 감지, 요약 API)  
- **AI:** OpenAI API로 알림용 2~3문장 요약 생성 (소형 모델 중심)

---

## 🚀 MVP 범위
- Watchlist 1~3개 (키워드·토픽 선택)  
- 소스: **Hacker News / arXiv RSS / GitHub Trending**  
- **룰 기반 탐지:** 키워드 + 단기 급증(velocity)  
- **알림 방식:** 푸시·이메일 + 앱 내 로그  
- **민감도 슬라이더:** 정확도 우선 / 범위 확장  
- 하루 4~6회 배치 수집 (비용 최소화)

---

## 💰 예상 비용
- **개발비:** 약 ₩5,000,000 (1~2주 집중 개발 기준)  
- **유지비:** ₩50,000 ~ ₩90,000/월  
  - Cloud Run 최소 인스턴스 0  
  - Firestore / FCM 무료 구간 활용  
  - SendGrid 무료 티어  
  - OpenAI 소형 모델 + 토큰 제한

---

## ⚠️ 리스크 & 유의사항
- 오탐률 상승 시 → 사용자 이탈 위험 → **고정밀 큐레이션 중심 운영**  
- 소스 정책 변경 시 파이프라인 수정 필요  
- 알림 과다 시 피로도 증가 → 기본값은 **저빈도·고정밀**  
- 실시간 집착 X → **수 시간 단위의 ‘의미 있는 속도’** 유지
