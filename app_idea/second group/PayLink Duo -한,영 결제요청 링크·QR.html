PayLink Duo -한,영 결제요청 링크·QR

GEMINI.md 베이스 · RN/Expo + Firebase + Gluestack UI · 한/영 동시 타겟

앱 개요 및 핵심 기능

  • 핵심 JTBD: “품목/금액/통화만 넣으면 한글·영문 결제요청 페이지(링크/QR)가 즉시 생성되어 메신저·메일로 공유해 입금 안내를 깔끔하게 보내고 싶다.”
  • 최소 기능:
    1. 요청 생성 폼: 품목명, 금액, 통화(KRW/USD), 수량, 마감일, 메모(선택).
    2. 한/영 자동 페이지: 금액·마감일·계좌/결제수단 안내를 KR/EN 병기로 렌더.
    3. 공유 수단: 앱 링크/짧은 URL(Firebase Dynamic Links) + QR 코드(이미지 저장/공유).
    4. 결제수단 프리셋: 국내 계좌/토스/카카오페이 링크, 해외용 PayPal.me 등 외부 링크 연결.
    5. (옵션) 상태 표시: “요청 발송/입금 확인” 수동 토글, 간단 메모.

기술 스택

  • Expo/RN: 빠른 배포(EAS), 권한 최소.
  • 라이브러리: react-native-qrcode-svg(QR), i18next(다국어), dayjs(날짜/타임존), expo-sharing, expo-clipboard.
  • Firebase: Auth(익명/Google), Firestore(요청 메타), Storage(QR 이미지 선택), Dynamic Links(짧은 URL), Analytics/Remote Config, Functions(서버렌더/클릭 로깅).
  • UI — Gluestack UI: Card(미리보기), Input/Select(폼), Badge(통화/상태), Button(공유/복사), Sheet(결제수단).
  • MCP 활용: 입력값→KR/EN 안내문 자동 생성(정중/캐주얼), 통화/날짜 지역 규칙 변환, 스토어 ASO 카피 다국어 생성.

수익 모델

  • 무료+광고: 요청 페이지 하단 배너, 월 20건 생성 제한.
  • 구독(월 $2.49): 광고 제거, 브랜딩/로고, 커스텀 도메인 프리픽스, 템플릿 저장, 다국 통화.
  • IAP: 1회 브랜딩 제거(요청 10건 팩), 맞춤 문구 템플릿 팩.

시장성 (왜 지금?)

  • 한국: 인스타/중고거래/DM 판매 확산 → 견적·입금 안내를 빠르고 깔끔하게 보내려는 니즈.
  • 글로벌: 한국 판매자 ↔ 해외 구매자(또는 반대) 간 양방향 KR/EN 안내 필요.
  • 실행 용이: 실제 결제 처리 없이도 링크·QR + 안내 문구만으로 가치 제공 → 1인 개발에 적합.
