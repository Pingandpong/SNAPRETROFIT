# QuietJournal — 로컬 AI 일기 앱 (Privacy-first)

## 1. 앱 콘셉트와 핵심 목표
### 콘셉트
- 모든 생성형 AI 기능이 *휴대폰 안에서* 끝나는 **프라이버시 중심 일기 앱**
- 제목 추천, 하루 요약, 다음 문장 제안 등 **완전 오프라인 AI**

### 핵심 목표
- “빠름 · 오프라인 · 안심”을 제공하는 체감형 일기 경험
- 클라우드 사용은 *필요할 때 유저 명시 동의 후*

## 2. 간단한 기술 스택
- **프론트:** React Native / Expo (OTA로 모델·프롬프트 빠른 업데이트)
- **로컬 AI:** 경량·정량화 LLM(텍스트 생성/요약), 기기 내 STT(선택)
- **백엔드(옵션):** Firebase(Auth, Firestore/Storage 백업), Cloud Run(API 라우팅, 결제 훅)
- **외부 API(옵션):** OpenAI API(고품질 요청에 한해 크레딧 소모)

## 3. MVP에서 우선 구현할 기능
### (1) 로컬 텍스트 어시스턴트
- 제목 자동 추천  
- 하루 요약  
- 다음 문장 제안  
→ **완전 오프라인**

### (2) 오프라인 초안 보관
- 모든 일기/사진/음성 로컬 저장  
- 나중에 *수동 백업 가능*

### (3) 프라이버시 대시보드
- 데이터가 로컬/클라우드인지 표시
- 클라우드 전송 기록 로그 제공
- 백업 On/Off
- 데이터 전체 삭제

### (4) 간단한 크레딧/쿼터
- 일일 무료 로컬 사용량
- 클라우드 고급 요청 시 유저 크레딧 차감

## 4. 선택 기능(2차 개발)
- 로컬 음성→텍스트(STT) 일기 작성
- 테마/감정 태그 자동 생성

## 5. 예상 총 개발 비용(1인 개발 기준)
### 개발 기간
- **6~8주**

### 비용
- 약 **₩1,500,000 내외**

### 초기 유지비(10~100명 기준)
- **₩50,000 이하**
  - Firebase 무료 구간 → 0원
  - Cloud Run 스케일-투-제로 → 거의 0원
  - OpenAI API → 월 1~3만 원 수준

## 6. 왜 이 구성이 저비용인가?
- **로컬 모델 처리** → 서버, GPU 비용 거의 없음
- **클라우드 요청 유료 크레딧 기반** → 변동비 상쇄
- **RN/Expo/Firebase/Cloud Run 조합** → 관리 포인트 최소

## 7. 잠재 리스크 및 유의사항
- **저사양 기기 지연** → “저전력 모드/충전 중 고성능” 옵션 제공
- **저장 공간 증가** → 경량 모델 기본, 고급 모델은 선택 다운로드
- **치료/상담 대체 아님 고지**
- **프라이버시 신뢰 확보 필수**

## 8. 간단한 개발 로드맵
### 주 1–2
- 기본 일기 UI
- 로컬 저장
- 온보딩

### 주 3–4
- 로컬 LLM 연동 (요약/제목/프롬프트)
- 프라이버시 대시보드 구현

### 주 5
- 크레딧/쿼터
- IAP 결제

### 주 6
- Cloud Run + OpenAI API 연동
- 베타 테스트
