최종 추천: SnapCard — 한 번 찍고, 바로 연락처 저장하는 네트워킹 도우미

앱 개념 & 핵심 목표

비즈니스 카드/배지/QR을 카메라로 찍으면, 이름·직함·회사·이메일·전화가 자동 추출되고 즉시 연락처로 저장

한 번의 확인으로 끝나도록 “캡처 → 빠른 검토 → 저장/팔로업” 흐름을 극단적으로 단순화

기본 오프라인·프라이버시 우선(사진·텍스트는 로컬 처리), 필요 시에만 클라우드 사용

간단 기술 스택

앱: React Native / Expo (Camera, Contacts, Permissions, EAS 빌드)

데이터: 로컬 저장(AsyncStorage/SQLite) + 선택적 Firebase Auth/Firestore(백업/동기화, 옵트인)

백엔드: Cloud Run(옵션, 회사 정보 보강이나 링크 미리보기 같은 부가기능용)

AI: on-device OCR 우선(ML Kit/TFLite) + OpenAI API(옵션, 팔로업 메시지 초안 생성용. 사용량 제한)

MVP에서 먼저 만들 것

핵심 1: “명함 스캔 → 연락처 저장” 단일 플로우(오프라인 동작, 한 화면에서 필드 검토/수정)

핵심 2: 저장 직후 1회성 “팔로업 메시지 템플릿” 제안(옵션, 비용절감을 위해 로컬 규칙 기반부터)

핵심 3: 최근 스캔 히스토리와 되돌리기(Undo), 중복 연락처 감지

나중에: 이벤트/장소 컨텍스트 태깅, 회사 정보 보강(Cloud Run), 메시지 템플릿 고급화(OpenAI API 소량)

예상 총 개발비(1인 개발 가정)

약 ₩3,000,000 (개발자 계정, 테스트 단말, 디자인/아이콘 등 실비 포함 기준)

월 유지비(저비용 운영)

약 ₩30,000/월 이하

Firebase 무료/저요금 티어 + Cloud Run 무부하 시 사실상 0원대

OpenAI API는 기본 비활성/사용량 제한으로 비용 통제(사용자 선택 시만 호출)

잠재 리스크/주의점

OCR 정확도: 디자인이 화려한 명함에서 필드 분류 오류 가능 → “검토/수정” 단계를 기본 제공

개인정보/권한: 카메라·연락처 권한 이유를 명확히 설명하고, 기본 로컬 처리 원칙 유지

비용 폭주: OpenAI 호출 남용 시 비용 증가 → 초안 생성은 무료 로컬 템플릿 우선, 고급 초안은 토글+쿨다운

경쟁: iOS/Android 기본 기능·타 스캔 앱과의 차별화 필요 → “저장까지 한 탭”, “팔로업 즉시 제안”, “오프라인/프라이버시”를 전면에

운영 팁(저비용 유지)

초기엔 온디바이스 OCR만으로 출시, 클라우드 보강은 과금 또는 옵트인

Firebase는 백업/동기화 요청 시에만 활성화(기본 로컬)

OpenAI는 월 무료 크레딧 한도 내에서만 제공하고 초과는 비활성 또는 소액 구독 연동
