최종 제안: WaterNow — 원탭 식물 물주기 스케줄러

앱 컨셉·핵심 목표
- 딱 “물주기”에만 집중한 초간단 앱. 사진 한 장 찍고, 식물 템플릿을 고르면, 자동으로 물주기 알림을 잡아줍니다.
- 복잡한 지식·AI 진단 없이 “기록 → 알림 → 완료”의 빠른 루틴을 만드는 것이 목표입니다.

기술 스택 (속도·효율 최적화)
- Expo Router + NativeWind: 카메라 화면, 알림, 딥링크까지 한 번에. 스타일링도 빠르게.
- Supabase: 인증/동기화/저장소를 한 곳에서. 초기엔 로컬 우선, 이후에 동기화(옵트인)만 붙이면 끝.

MVP에서 먼저 만들 것
- 카메라 캡처 + 식물 템플릿(다육이/허브/포토스 등 기본 주기 포함)
- 로컬 물주기 알림 스케줄러(원탭 스누즈, 알림 → 상세 딥링크)
- ‘Watered’ 원탭 기록 + 아주 짧은 히스토리 카드(마지막 물 준 날짜)
- 이미지 경량화(클라이언트 리사이즈)와 로컬 저장 기본, 동기화는 옵트인
- 5개 식물 제한(향후 프리미엄 확장 여지)

잠재 리스크·주의점
- 알림 신뢰성(특히 Android 백그라운드) 확보 필요
- 기능이 매우 단순해 수익화가 약할 수 있음 → “무제한 식물/동기화/백업”을 프리미엄으로 준비
- 사진 업로드는 옵트인으로 두고, 이미지 압축·보존정책으로 저장소 비용 관리

왜 지금 이 스택인가?
- Expo + NativeWind로 화면/알림/카메라를 일주일 내 완성 가능, 디자인 속도도 빠름
- Supabase로 인증·DB·스토리지를 한 번에 붙여 서버 작업을 최소화
- 로컬 우선 → 동기화(옵트인) 구조라 초기에 비용이 매우 낮고, 확장도 쉬움
