JD Snap - 채용 공고 스샷 맞춤 지원 문장 생성

GEMINI.md 베이스 · RN/Expo + Firebase + Gluestack UI · 한/영 동시 타겟

앱 개요 및 핵심 기능

  • 핵심 JTBD: “채용공고를 캡처/붙여넣기만 하면, 요구 역량 키워드를 뽑고 내 이력에 맞춘 한/영 지원 문장을 바로 얻고 싶다.”
  • 최소 기능:
    1. 공고 입력: 스크린샷 OCR 또는 텍스트 붙여넣기.
    2. 키워드 추출: 역량/툴/연차/직무 자동 태깅.
    3. 맞춤 문장 생성(한/영): 프로젝트·역할·성과 선택 → 지원 문장 생성.
    4. 매칭 점수: 요구 키워드 대비 보유 항목 비율(간단 규칙 기반).
    5. 내보내기: 클립보드/이메일/노션 공유(한·영 동시 출력).

기술 스택

  • RN/Expo: 크로스플랫폼/EAS 빌드.
  • 입력/미디어: expo-image-picker, (촬영 필요 시) react-native-vision-camera
  • OCR: Google ML Kit Text Recognition v2(네이티브 브리지).
  • 상태/폼: zustand, react-hook-form, react-navigation, dayjs
  • Firebase: Auth(익명/Google), Firestore(세션/히스토리), Analytics, Remote Config, Crashlytics
  • UI(Gluestack): Card(JD 미리보기), Badge(키워드), Sheet(문장 옵션), Tabs(KR/EN)
  • MCP: OCR 텍스트 → 공고 JSON 구조화(회사/직무/자격/우대/스택), STAR 템플릿 한·영 문장 생성, ASO 카피 자동화

수익 모델

  • 무료+광고: 결과 화면 하단 배너(생성 과정엔 미노출).
  • 구독(월 $3.99): 광고 제거, 한/영 무제한 문장, 키워드 사전 저장, 템플릿 커스텀.
  • IAP: 1회 패스(문장 10회), 커버레터 전체 초안 패키지.

시장성 (왜 지금?)

  • 한국: 이직·취업 준비 증가, 공고별 키워드 맞춤과 영문 문장 작성 부담 큼.
  • 글로벌: JD가 길고 포맷 제각각 → “TL;DR + 맞춤 문장” 니즈 상시 존재.
  • 기술: 온디바이스 OCR + 경량 LLM로 앱 내 즉시 결과 제공 → 초기 비용↓, 1인 개발 적합.
