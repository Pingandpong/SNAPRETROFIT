Half&Meet — 두 사람이 만날 ‘중간지점’ 추천

RN/Expo + Firebase · KR/EN 동시 타겟 · 1인 개발 즉시 출시용 MVP

앱 개요 및 핵심 기능

  • 핵심 JTBD: “두 위치를 넣으면 공평한 중간 후보를 카드 형태로 보여주고, 메신저로 바로 공유하고 싶다.”
  • 최소 기능:
    1. 두 위치 입력(현위치 + 주소 붙여넣기) & 카테고리 선택(카페/식사/디저트/조용한).
    2. 후보 10개 카드: 양측 평균 이동시간, 평점, 가격대 아이콘, 영업중 여부.
    3. 1탭 공유: 카톡/왓츠앱용 이미지/텍스트 카드(KR/EN).
    4. 즐겨찾기·최근 기록 저장.
    5. 길찾기 딥링크: Google · Naver · Kakao 지도 열기.

기술 스택

  • Expo/RN: expo-location (현위치), react-native-maps (지도/마커), @tanstack/react-query (API 캐싱), react-hook-form (폼), expo-linking (지도 딥링크), expo-sharing (카드 공유), react-native-svg (미니 배지/아이콘)
  • Firebase: Anonymous Auth, Firestore(즐겨찾기/최근), Analytics, Remote Config(A/B), (선택) Functions(API 키 프록시)
  • Gluestack UI: Card, Button, Select, Badge, Tabs, Skeleton, Toast
  • MCP 활용: 시간대/날씨/카테고리로 추천 문구 생성, 상위 후보 장단점 요약, 스토어 ASO 카피 자동화

수익 모델

  • 무료+광고: 결과 리스트 하단 배너, 1세션 2회 이하 제한.
  • Pro 구독(월 ₩2,900): 광고 제거, 무제한 추천/즐겨찾기, 고급 필터(영업중/프랜차이즈 제외/가격대), 데이트 카드 테마.
  • IAP: 프리미엄 카드 테마 팩(계절/미니멀/러블리).

시장성 (왜 지금?)

  • 한국: 데이트·스터디·점심 미팅에서 “중간에서 보자” 니즈 고정. 기존 지도앱은 두 출발지 기준 중간 후보를 원탭으로 추천하지 않음.
  • 글로벌: 리모트 근무·외곽 거주 증가로 공평 이동 수요 확대. 웹 도구는 있으나 한국어/모바일 네이티브 경험 약함.
  • 실행 용이: Google Places/Directions 기반 랭킹(거리 편차·평점·영업중)으로 1–2주 MVP 가능.

주요 경쟁 앱 및 차별점

  • Google Maps — 길찾기 강점이나 두 출발지 중간 후보 자동 추천 부재 → 중간지점 전용 과업 특화.
  • KakaoMap / Naver Map — 로컬 강점이나 공평 추천·공유 카드 경험 약함 → 1탭 공유·KR/EN 동시.
  • Whatshalfway(웹) — 간단 웹 도구, 모바일 네이티브/카톡 공유 최적화 부족 → 네이티브 공유·딥링크 강점.
  • Between — 커플 커뮤니케이션 중심 → 만남 장소 결정 문제에 집중.
  • Meet Halfway류(스토어 다수) — 지역화/업데이트 미흡 사례 → 한국어 우선·데이트 카드 테마로 차별화.
