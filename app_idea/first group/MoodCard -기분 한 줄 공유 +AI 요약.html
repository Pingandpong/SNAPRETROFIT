MoodCard — 한 줄 감정 기록 + AI 요약

RN/Expo + Firebase · KR/EN 동시 타겟 · 1인 개발 즉시 출시용 MVP

앱 개요 및 핵심 기능

  • 핵심 JTBD: “오늘 기분을 한 줄로 남기면 AI가 감정/요약 카드를 만들어주고, 공유까지 바로 하고 싶다.”
  • 최소 기능:
    1. 텍스트 입력(선택: 30초 이내 음성 → STT).
    2. 감정 분류(긍·부·중립 + 키워드) & 한/영 한 줄 요약.
    3. 카드 저장/리스트(즐겨찾기).
    4. 공유(인스타 스토리, 카톡 이미지/텍스트, expo-sharing).
    5. 주간 미니 그래프(스파크라인).

기술 스택

  • Expo/RN: expo-av (음성 선택), expo-file-system, expo-sharing, react-native-svg (차트)
  • Firebase: Auth(익명), Firestore(카드/감정 라벨), Storage(공유 이미지 선택), Analytics/Remote Config
  • UI (Gluestack UI): Card(결과), Badge(감정), Tabs(KR/EN), Button(공유)
  • MCP 활용: 입력 문장 → 감정 라벨·요약 프롬프트, 공유용 카피 및 ASO 문구 자동 생성(KR/EN)

수익 모델

  • 무료+광고: 최근 7개 카드 저장, 리스트 하단 배너.
  • 구독(월 ₩3,500): 광고 제거, 무제한 저장, 고급 감정 라벨(세부 감정 6~8종), CSV 내보내기.
  • IAP: 카드 테마/폰트 팩.

시장성 (왜 지금?)

  • 한국: Z세대 중심 짧은 기록·셀프케어 관심 ↑, “길게 쓰기 부담” → 한 줄 입력 선호.
  • 글로벌: 일기/감정 앱 다수이나 영어 위주·세팅 복잡 → 경량·이중언어 틈새.
  • 실행 용이: 텍스트/짧은 음성 → DB 저장 → AI 감정 요약 카드만으로 초기 가치 제공.

주요 경쟁 앱 및 차별점

  • Daylio — 아이콘 기반 분위기/활동 로그 중심 → 자유 텍스트 한 줄 + 즉시 AI 요약 카드.
  • Stoic — 영어 중심 저널링·루틴 풍부 → KR/EN 이중언어·초경량 입력에 집중.
  • Reflectly — 일기 동기부여 강점, 가입 플로우 상대적으로 길다 → 무가입/익명 시작으로 즉시 기록.
  • Journey — 다기능 저널 플랫폼 → 공유용 카드 생성·숏폼 친화 포맷 특화.
  • Presently — 감사 일기 특화 → 감정 전반 + 한/영 요약 제공.
